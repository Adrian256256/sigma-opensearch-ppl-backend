# ECS Field Mapping for Sigma Rules

## Overview

This directory contains tools and data for mapping Sigma rule fields to the Elastic Common Schema (ECS) standard. The goal is to ensure compatibility between Sigma rules and ECS-compliant SIEM platforms like OpenSearch.

---

## Files

### Scripts

#### `checker.py`

A Python script that extracts all field names from Sigma rules in the official [SigmaHQ repository](https://github.com/SigmaHQ/sigma).

#### What it does:
1. **Recursively scans** all Sigma rule files (`.yml` and `.yaml`) in the `sigma-master/rules/` directory
2. **Parses** the `detection` section of each rule
3. **Extracts** field names, including:
   - Plain field names (`CommandLine`, `Image`)
   - Fields with modifiers (`CommandLine|contains` â†’ extracts `CommandLine`)
   - Nested fields from complex detection logic
4. **Tracks** which rule each field was extracted from
5. **Generates GitHub links** to each rule in the official Sigma repository
6. **Outputs** results to two CSV files:
   - `sigma_fields.csv` - unique fields only (deduplicated and sorted)
   - `sigma_fields_with_paths.csv` - all field-rule pairs with source paths and GitHub links

#### Usage:
```bash
python3 checker.py
```

#### Output:
- **`sigma_fields.csv`**: Contains all unique field names found across all Sigma rules (one field per line)
- **`sigma_fields_with_paths.csv`**: Contains field names with their source rule paths and direct GitHub links (multiple entries per field showing all rules using that field)

#### `add_ecs_links.py`

A Python script that enhances the `ecs_verification_results.csv` file by adding direct links to the official ECS documentation for each field.

##### What it does:
1. **Reads** the existing `ecs_verification_results.csv` file
2. **Extracts** the field set from each ECS field (`process` from `process.command_line`)
3. **Generates** direct links to the ECS documentation for each field set
4. **Updates** the CSV file with a new `ecs_documentation_link` column

##### Usage:
```bash
python3 add_ecs_links.py
```

##### Output:
Updates `ecs_verification_results.csv` in-place with ECS documentation links pointing to:
- `https://www.elastic.co/guide/en/ecs/current/ecs-{field_set}.html`

##### Examples:
- `process.command_line` â†’ `https://www.elastic.co/guide/en/ecs/current/ecs-process.html`
- `file.path` â†’ `https://www.elastic.co/guide/en/ecs/current/ecs-file.html`
- `event.action` â†’ `https://www.elastic.co/guide/en/ecs/current/ecs-event.html`

#### `count_field_frequency.py`

A Python script that analyzes Sigma field usage frequency across all rules and generates a sorted frequency report.

##### What it does:
1. **Reads** the `sigma_fields_with_paths.csv` file
2. **Counts** how many times each field appears across all Sigma rules
3. **Sorts** fields by frequency (most common first)
4. **Outputs** results to `sigma_fields_frequency.csv`

##### Usage:
```bash
python3 count_field_frequency.py
```

##### Output:
- **`sigma_fields_frequency.csv`**: Contains all unique fields with their occurrence count, sorted by frequency (descending)

##### Statistics (as of latest run):
- **363 unique fields** across **6,539 total field occurrences**
- Top 3 most frequent fields: `Image` (1,341), `CommandLine` (1,076), `OriginalFileName` (561)

### Data Files

#### `sigma_fields.csv`


#### Source:

#### `sigma_fields_with_paths.csv`

A CSV file containing all Sigma field names along with the path to the rule file where each field was found, plus direct GitHub links to view each rule.

#### Structure:
```csv
field,rule_path,github_link
CommandLine,windows/process_creation/proc_creation_win_susp_cmd.yml,https://github.com/SigmaHQ/sigma/blob/master/rules/windows/process_creation/proc_creation_win_susp_cmd.yml
CommandLine,windows/process_creation/proc_creation_win_powershell.yml,https://github.com/SigmaHQ/sigma/blob/master/rules/windows/process_creation/proc_creation_win_powershell.yml
Image,windows/process_creation/proc_creation_win_susp_binary.yml,https://github.com/SigmaHQ/sigma/blob/master/rules/windows/process_creation/proc_creation_win_susp_binary.yml
...
```

#### Columns:
- **`field`**: The Sigma field name
- **`rule_path`**: Relative path to the rule file (from `sigma-master/rules/`)
- **`github_link`**: Direct link to view the rule on GitHub

#### Source:
- Generated by `checker.py` from the [SigmaHQ/sigma](https://github.com/SigmaHQ/sigma) repository
- Contains multiple entries per field (one for each rule that uses it)
- Sorted alphabetically by field name, then by rule path
- GitHub links point to the master branch of the official Sigma repository

#### `ecs_verification_results.csv`

A comprehensive mapping of Sigma fields to their ECS equivalents with confidence levels and direct links to ECS documentation.

#### Structure:
```csv
sigma_field,ecs_field,ecs_documentation_link,notes
Message,message,https://www.elastic.co/guide/en/ecs/current/ecs-base.html,High confidence - Official ECS core field
CommandLine,process.command_line,https://www.elastic.co/guide/en/ecs/current/ecs-process.html,High confidence - Full command line
AccessMask,winlog.event_data.AccessMask,https://www.elastic.co/guide/en/ecs/current/ecs-base.html,High confidence - Windows access mask
...
```

#### Columns:
- **`sigma_field`**: The original Sigma field name
- **`ecs_field`**: The recommended ECS field(s) to use
  - Single field for direct mappings (`user.name`)
  - Multiple fields separated by `|` for context-dependent mappings (`source.address | destination.address`)
- **`ecs_documentation_link`**: Direct link to the official ECS documentation for the field set
- **`notes`**: Confidence level and explanation
  - **High confidence**: Direct match with official ECS fields
  - **Medium confidence**: Logical mapping but may require context or validation
  - **Low confidence**: Generic field that needs contextual interpretation

#### Tools:
- **`add_ecs_links.py`**: Script to add ECS documentation links to the CSV
  ```bash
  python3 add_ecs_links.py
  ```
